# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in OpenPresenter, **please do not open a public issue.**

Instead, report it privately using GitHub's security advisory workflow:

1. Go to the repository's **Security** tab.
2. Click **Report a vulnerability** (this opens a private draft advisory visible only to the maintainers).
3. Provide a clear description, steps to reproduce, and any relevant logs or proofs of concept.

The maintainers will acknowledge the report, work with you on a fix, and coordinate a disclosure timeline. You will be credited for the discovery unless you prefer to remain anonymous.

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

Only the latest released minor version receives security fixes.
